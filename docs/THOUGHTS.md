# 核心思考与设计哲学 (Core Thoughts & Design Philosophy)

本项目不仅仅是一个文献查询工具，它是一个“知识转化引擎”，旨在解决传统医学研究中的碎片化与门槛问题。以下是关于黄精研究探索器（Huangjing Research Explorer）的核心思考。

## 1. 为什么选择黄精？ (Why Huangjing?)
黄精是“药食同源”的典型代表。它既有深厚的文化底蕴（“九蒸九晒”），又有现代科学的验证（多糖、甾体皂苷）。它是连接“养生文化”与“药理研究”的最佳基点。我们通过这个项目，试图回答一个问题：**如何让非专业的养生爱好者也能读懂复杂的药理实验结果？**

## 2. 设计哲学：从“原始数据”到“行动指南” (Raw Data to Actionable Insights)
在数据处理上，我们将信息分为三个维度：
- **深度数据层 (Full Database)**: 1100+ 条原始文献。这是专业研究者的“大库”，保证了系统的权威性。
- **代表文献层 (Representative Gallery)**: 精选的 50 篇具有里程碑意义的文献。这是系统的“策展部分”，为用户指明研究重点。
- **知识应用层 (FAQ & Synthesis)**: 100+ 个问答。这是将上述数据“反刍”后的产物，直接回答用户的日常疑惑。

## 3. 技术驱动：WASM 与 极致体验 (Tech-Driven UX)
我们选用了 **DuckDB-Wasm** 和 **SQL.js**。这种“浏览器内数据库”的技术选型不仅仅是为了炫技，而是为了实现：
- **秒级检索**: 即使在 1100+ 条记录中搜索，也能实现即时响应，无需经过缓慢的 API 往返。
- **隐私保护**: 所有搜索逻辑都在用户的本地设备上完成，不收集搜索意图。

## 4. 视觉语言：新丑风 (Neubrutalism) 与 科学感 (Scientific Aesthetic)
本项目的 UI 采用了大色块、粗边框的“新丑风”，并结合了精密仪器的“科学感”。
- **新丑风**: 象征着传统医学的质朴与直白。
- **科学感**: 象征着现代研究的严谨与精确。
这种视觉冲撞旨在告知用户：**我们既尊重传统，也不迷信盲从。**
